Programa voltado para verificar a rota tomada por um determinado pacote.
Nem todo `traceroute` binário possui as mesmas funcionalidades.

---

## Instalação

```bash
sudo apt install traceroute
sudo apt remove inetutils-traceroute
```

```bash
$ traceroute -V

```

## Uso Básico

Uso básico do traceroute:

```bash
traceroute www.alvo.com
```

:::info
Ele utiliza o padrão UDP.
:::

Para alterar o valor do TTL:

```bash
traceroute www.alvo.com -m 1
```

:::info
Nesse caso haverá apenas um salto.
:::

Para ir para um salto específico:

```bash
traceroute www.alvo.com -f 8
```

Para não resolver o nome e exibir apenas o IP:

```bash
traceroute www.alvo.com -n
```

Para enviar pacotes ICMP:

```bash
traceroute www.alvo.com -I
```

Para enviar pacotes TCP:

```bash
traceroute www.alvo.com -T
```

Mostrar a AS (Autonomous System):

```bash
traceroute www.alvo.com -A
```

Utilizando UDP, padrão porta 53:

```bash
traceroute www.alvo.com -U
```


- Por padrão sempre são enviados 3 pacotes e podemos ver a latência de cada um com
  sucesso em cada hop.
- No *hop* 3 e 5, falhou todas as tentativas, podendo ser timeout ou firewall bloqueando.
  Para validar se é firewall mude o protocolo entre UDP, TCP (teste mais de uma porta) e ICMP.
- No *hop* 10 há vários IP, isso significa a presença de um *loadbalancer*.
- No *hop* 6, o asterísco no início ou no final (hop 4 ou 8), indica que falharam
  por timeout.
